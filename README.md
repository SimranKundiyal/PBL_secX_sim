INTRODUCTION 

An end-to-end machine learning system designed to preprocess campus Wi-Fi data, perform exploratory data analysis (EDA), and apply feature selection techniques to build classification models for accurate prediction of network congestion.

Question Statement --> Design and implement a complete machine learning classification system to analyze and predict congestion in campus Wi-Fi networks using network performance data. The project involves selecting an appropriate dataset containing numerical attributes such as the number of connected devices, network latency, packet loss rate, bandwidth usage, and access point load. Data preprocessing steps like data cleaning, normalization, and transformation are carried out to maintain data consistency and quality. Exploratory Data Analysis is performed to examine the distribution of network metrics, visualize patterns in network behavior, identify congestion intervals, detect anomalies, and study relationships between different parameters using statistical methods and visual tools. Correlation analysis is used to evaluate how various features influence congestion. Based on the insights obtained from EDA, important features are selected (using methods like PCA, Forward Selection, Backward Selection) to enhance model performance. Machine learning classification models are then developed to predict network congestion using suitable algorithms. The trained models are evaluated using standard performance metrics (such as accuracy) to measure their effectiveness, helping in better network management and decision-making.

🔍 Phase 1: Data Cleaning & Exploratory Data Analysis

In this phase, the dataset was initially loaded and analyzed to understand its structure, feature types, and overall composition. Data cleaning techniques were applied to handle missing values, eliminate duplicate records, and correct inconsistencies to ensure high data quality. After preprocessing, exploratory data analysis (EDA) was carried out using statistical summaries and visualizations to identify patterns, distributions, and possible outliers. This step provided a clear understanding of the dataset and highlighted key relationships between variables, which helped in guiding further processing and modeling steps.

⚙️ Phase 2: Feature Engineering & Preprocessing

This phase focused on converting the cleaned dataset into a machine learning–ready format. Features were examined and grouped based on their importance, and correlation analysis was performed to understand their impact on the target variable (congestion). Outliers were identified and treated to improve data reliability. Categorical variables (if any) were encoded, and significant features were selected to reduce unnecessary noise. Additionally, feature scaling methods were applied to standardize the data, and dimensionality reduction techniques such as PCA were used to improve efficiency while retaining important information.

🤖 Phase 3: Model Building & Evaluation

In the final phase, the processed dataset was divided into training and testing sets to ensure proper evaluation of the model. Multiple machine learning models were implemented and trained on the prepared data. These models learned patterns from the input features and were evaluated using appropriate performance metrics such as accuracy and other scores. A comparison of different models was performed to identify the most effective one. This phase demonstrated the predictive capability of the system and showed how preprocessing and feature engineering contributed to improving overall model performance.
